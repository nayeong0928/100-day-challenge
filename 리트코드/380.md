# 380. Insert Delete GetRandom O(1)

## 문제 분석

- insert(val) : 값이 있으면 false 리턴, 값이 없으면 val을 삽입하고 true 리턴
- remove(val) : 값이 없으면 false 리턴, 값이 있으면 val을 삭제하고 true 리턴
- getRandom() : 원소들을 같은 확률로 랜덤하게 리턴함, 이 메소드가 호출될 때에는 최소한 1개의 원소가 있음
- -2^31 <= val <= 2^31 - 1

## 풀이

### ArrayList와 HashMap을 선택한 이유

- ArrayList는 랜덤 리턴을 위해서 필요
- HashMap에 key로 값을 넣고, 배열에서의 인덱스를 value로 넣어서 변경사항을 ArrayList에서도 동기화 할 수 있도록 함
- ArrayList의 경우 삭제할 때에는 시간 복잡도가 O(N) 이라는 단점을 해결하기 위해서 요소를 삭제할 때
    1. 삭제할 인덱스 자리에 배열의 맨 마지막 값 덮어쓰기
    2. 맨 마지막 요소 삭제
    -> 이 방법을 사용하면 시간복잡도 O(1) 로 해결 가능하다.


## 전체코드 

``` java

class RandomizedSet {

    static HashMap<Integer, Integer> map;
    static ArrayList<Integer> list;
    static int check;

    public RandomizedSet() {
        map=new HashMap<>();
        list=new ArrayList<>();
    }
    
    public boolean insert(int val) {
        if(map.containsKey(val)){
            return false;
        }

        map.put(val, list.size()-1);
        list.add(val);
        return true;
    }
    
    public boolean remove(int val) {
        if(!map.containsKey(val)){
            return false;
        }

        int idx=list.indexOf(val);
        int lastVal=list.get(list.size()-1);
        list.set(idx, lastVal);
        list.remove(list.size()-1);
        map.remove(val);
        
        return true;

    }
    
    public int getRandom() {
       Random random=new Random();
       return list.get(random.nextInt(list.size()));       
    }
}
```
